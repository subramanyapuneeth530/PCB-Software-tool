# PCB Gerber Viewer Pro

A professional-grade, open-source PCB Gerber file viewer built in Python and PyQt6. Designed for hardware engineers, PCB designers, and manufacturing engineers who need fast, accurate inspection of Gerber fabrication files without a full EDA suite.

---

## Features at a Glance

| Category | Feature |
|---|---|
| **Formats** | RS-274X, Gerber X2, Excellon drill |
| **Connectivity** | Spatial grid index, flood-fill net tracing |
| **DRC** | Clearance check, min track width, configurable thresholds |
| **Rendering** | Per-layer pixmap cache, minimap, grid overlay, hover tooltips |
| **Export** | PNG, SVG |
| **UI** | Dark / Light / High Contrast themes, recent folders, settings persistence |
| **Analysis** | Board statistics, measurement tool, net name labels |

## Installation

**Requirements:** Python 3.10+ and PyQt6.

```bash
pip install PyQt6
```

**Run:**


To open a folder directly from the command line:


## Supported File Formats

### Gerber (RS-274X)

The industry standard copper, mask, silk, paste, and outline format generated by all major EDA tools including KiCad, Altium Designer, Eagle, EasyEDA, and OrCAD.

Supported commands and features:

- Format specification (`%FSLAX%`) — integer and decimal digit counts
- Unit modes — `%MOMM%` (millimetres) and `%MOIN%` (inches, auto-converted to mm)
- Aperture definitions — Circle (`C`), Rectangle (`R`), Oblong (`O`), Polygon (`P`)
- Draw commands — `D01` (draw), `D02` (move), `D03` (flash/pad)
- G codes — `G01` linear, `G02`/`G03` arc modes, `G36`/`G37` region fill
- Multi-block extended commands — handles KiCad's concatenated `%...%` tokens
- Track aperture snapshotting — width captured at track start, not at close

### Gerber X2 Attributes

Gerber X2 embeds component metadata directly in the file via `%TO.%` blocks. When your EDA tool exports with X2 enabled (KiCad does this by default), the viewer reads:

| Attribute | Block | What it carries |
|---|---|---|
| Net name | `%TO.N,GND*%` | Electrical net — GND, VCC, SDA, etc. |
| Component ref | `%TO.C,U1*%` | Reference designator — R1, C5, U2 |
| Pin number | `%TO.P,U1,3*%` | Pin number on that component |

These appear in the Info panel when you click a pad and as floating labels on the canvas when zoomed in past 8× scale.

If your files were exported without X2, a KiCad `.net` netlist can be loaded separately via **Analysis → Load Netlist**.

### Excellon Drill Files

Parsed and rendered as a dedicated overlay layer:

- Tool definitions with diameter (`T1C0.800`)
- Metric and imperial units with auto-detection
- Leading/trailing zero suppression
- Plated holes (silver) and non-plated holes (amber) distinguished visually
- Supported extensions: `.drl`, `.xln`, `.exc`, `.ncd`

## Layer System

### Auto-Detection

Layers are identified by filename pattern matching against all standard naming conventions:

| Layer | Default | Patterns recognised |
|---|---|---|
| Board Outline | Yes | `Edge.Cuts`, `Edge_Cuts.gbr`, `.GKO` |
| Front Copper | Yes | `F.Cu`, `F_Cu.gbr`, `.GTL` |
| Back Copper | Yes | `B.Cu`, `B_Cu.gbr`, `.GBL` |
| Inner 1–4 | Yes | `In1.Cu`–`In4.Cu`, `.G1`–`.G4` |
| Drill Holes | Yes | `.DRL`, `.XLN`, `.EXC`, `.NCD` |
| Front/Back Mask | On demand | `F.Mask`, `B.Mask`, `.GTS`, `.GBS` |
| Front/Back Silk | On demand | `F.Silkscreen`, `B.Silkscreen`, `.GTO`, `.GBO` |
| Front/Back Paste | On demand | `F.Paste`, `B.Paste` |
| Front/Back Fab | On demand | `F.Fab`, `B.Fab` |
| Front/Back Courtyard | On demand | `F.CrtYd`, `B.CrtYd` |

### Lazy Loading

Only copper layers, the board outline, and drill files load at startup. All other layers appear in the panel as available but are not parsed or rendered until you tick their checkbox. This keeps startup fast — a 4-layer board with silkscreen loads in under a second.

Unticking a layer fully unloads it from memory and rebuilds the connectivity graph.

## Net Tracing

Click any pad, track, or copper region to highlight its entire connected net:

- The clicked element turns white
- All connected copper turns gold
- Everything else dims to ~10% opacity
- The Info panel populates with element details
- The status bar reports the number of connected primitives

Connectivity uses a **spatial grid index**. The board is divided into 2 mm cells; only primitives sharing a cell are checked for physical overlap. This is O(n log n) versus the naive O(n²) all-pairs approach — roughly 100× faster on a typical 3000-primitive board. The build runs on a background `QThread` so the window stays responsive.

Right-click or press `Esc` to clear the selection.

---

## DRC (Design Rule Check)

**Analysis → Run DRC** checks two categories of violation:

- **Clearance** — unconnected copper elements that are closer together than the minimum clearance threshold. Overlapping unconnected copper is always flagged regardless of threshold.
- **Minimum track width** — tracks thinner than the configured minimum.

Violations are marked directly on the canvas — red `✗` for clearance errors, amber `⚠` for width warnings — and a full text report opens in a separate window.

**Analysis → DRC Settings** lets you configure both thresholds. Defaults: 0.10 mm clearance, 0.05 mm track width.

## Measurement Tool

Press `M` or click **Measure** in the toolbar:

1. Click the first point — a circle marker appears
2. Move the cursor — a live dashed ruler follows, showing the running distance
3. Click the second point — distance is locked and displayed on the canvas

The readout shows: total distance, ΔX, and ΔY in millimetres to 4 decimal places. Press `M` again or `Esc` to exit. Click again to start a new measurement.

## Themes

Three themes available from the toolbar dropdown or **View → Theme**:

- **Dark** — near-black background, amber accent. Default. Good for long sessions.
- **Light** — white background, blue accent. Good for printing or bright rooms.
- **High Contrast** — pure black and white with yellow accent. Maximum readability.

Theme is saved between sessions.

## Building a Standalone Executable

To distribute as a `.exe` on Windows (no Python required on the target machine):

```bash
pip install pyinstaller
pyinstaller build.spec
```

Output is `dist/PCBViewerPro/`. Zip and distribute. Typical bundle size is 80–120 MB due to the Qt runtime.

For a proper Windows installer with shortcuts and an uninstaller, pass the output folder through [Inno Setup](https://jrsoftware.org/isinfo.php).

For macOS, the same spec produces a `.app` bundle. Wrap with `create-dmg` for a disk image.

## What Gerber Can and Cannot Tell You

Gerber is a manufacturing format. It describes copper shapes, not schematic intent.

**What is readable directly from Gerber files:**

- Pad position (X, Y in mm) — from `D03` flash coordinates
- Pad shape and size — from the aperture definition
- Which layer the pad belongs to — from the filename
- Track width — from the circular aperture used to draw it
- Solder mask openings — `D03` flashes on the mask layer
- SMD vs through-hole — SMD pads appear on copper only; TH pads align with drill entries
- Net name, ref, pin number — **only if the file includes Gerber X2 `%TO.%` attributes**

**Not stored in Gerber — requires a netlist or native PCB file:**

- Component reference designators (R1, C5) — unless Gerber X2 is present
- Pin numbers — unless Gerber X2 is present
- Net names — unless Gerber X2 is present
- Component values (10 kΩ, 100 nF) — schematic / BOM only

## Recent Updates (Stability & Integration)

The software has undergone a rigorous cycle of integration testing against complex industry datasets (e.g. A64-OlinuXino, ESP32-S3 custom boards), identifying and fixing corner-case rendering and thread-lifecycle bugs:

- **Excellon Drill Patch**: The parser was refactored with a robust system to track imperial/metric formatting safely across both `00.0000` integers and direct decimal representation `X3.9469` bypassing regex stripping faults. 
- **Qt Engine Hotfixes**: Eliminated silent C++ UI lifecycle crashing bugs connected to label updates within the info panel overlay. Properly guarded QThread deallocation (`deleteLater()`) preventing Zombie background threads hanging operations post-close.
- **Headless Parser Utility**: For bulk diagnostic checks, a CLI analyzer automatically validates directory folders reporting total element extraction numbers independently from the core GUI wrapper.
